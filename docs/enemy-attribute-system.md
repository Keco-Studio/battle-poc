# 敌人属性系统设计文档

## 1. 敌人属性

### 基础属性
| 属性 | 说明 |
|------|------|
| HP   | 敌人生命值 |
| ATK  | 敌人攻击力 |
| DEF  | 敌人防御力 |
| SPD  | 敌人速度 |

### 基础字段
| 字段 | 类型 | 说明 |
|------|------|------|
| id | number | 敌人唯一ID |
| name | string | 敌人名称 |
| x | number | 地图X坐标 |
| y | number | 地图Y坐标 |
| level | number | 敌人等级 |
| profile | object | 可选属性覆盖配置 |

### 当前默认敌人
| ID | 名称 | 坐标 | 等级 |
|----|------|------|------|
| 1 | 恶魔守卫 | (5, 5) | 3 |
| 2 | 暗影刺客 | (10, 6) | 5 |

---

## 2. 等级与属性规则

### 敌人等级
- 地图中的敌人可直接配置固定等级
- 进入战斗时，也可根据玩家等级动态生成敌人等级
- 当前动态等级规则: `玩家等级 - 1` 或 `玩家等级 - 2`
- 敌人最低等级不低于 `1`

### 属性计算
- 敌人属性直接按固定基础值与成长值计算

### 敌人基础成长曲线
| 属性 | 1级基础值 | 每级成长 |
|------|-----------|----------|
| HP   | 120       | +36      |
| ATK  | 6         | +6       |
| DEF  | 3         | +3       |
| SPD  | 3         | +3       |

### 敌人属性公式
- `enemyMaxHp = 120 + (level - 1) * 36`
- `enemyAtk = 6 + (level - 1) * 6`
- `enemyDef = 3 + (level - 1) * 3`
- `enemySpd = 3 + (level - 1) * 3`

### 计算示例
以 `Lv.3` 敌人为例:

- 敌人 Lv.3 属性:
  - HP = `120 + (3 - 1) * 36 = 192`
  - ATK = `6 + (3 - 1) * 6 = 18`
  - DEF = `3 + (3 - 1) * 3 = 9`
  - SPD = `3 + (3 - 1) * 3 = 9`

## 3. 战斗相关规则

### 攻速规则
- 敌人与玩家共用同一套攻速换算公式
- 攻击间隔公式: `attackInterval = clamp(1150 - SPD * 28, 380, 2200)`
- SPD 越高，攻击间隔越短

### 承伤规则
- 当前战斗采用平滑护甲减伤，而不是简单的 `攻击 - 防御`
- 公式如下:

`有效伤害 = rawDamage * K / (K + armor)`

- 其中 `K = 50`
- 最终伤害至少为 `1`

### 行为规则
- 当前脚本AI仅控制敌方单位
- 敌方默认行为为:
  1. 查找第一个存活的玩家目标
  2. 执行普通攻击
- 当前未定义敌人专属技能逻辑

---

## 4. 经验关系

### 经验来源
- 玩家击败怪物后获得经验
- 角色设计文档中的经验规则为: `战胜怪物获得 = 怪物等级`

### 关系说明
- 敌人等级越高，玩家获得经验越多
- 当前敌人属性系统与玩家经验系统通过“敌人等级”关联

## 5. 数据结构

```typescript
interface Enemy {
  id: number;
  name: string;
  x: number;
  y: number;
  level: number;
  profile?: {
    maxHp?: number | null;
    atk?: number | null;
    def?: number | null;
    spd?: number | null;
  };
}

interface EnemyCombatStats {
  maxHp: number;
  atk: number;
  def: number;
  spd: number;
}
```

### 说明
- `Enemy` 用于地图配置与敌人生成
- `EnemyCombatStats` 用于进入战斗后的敌人数值
- `profile` 为预留字段，可用于手动覆盖默认成长结果

---

## 6. 当前实现总结

1. 敌人属性按独立的基础值与成长值计算
2. 敌人数值当前采用固定成长曲线
3. 敌人战斗行为目前较简单，默认只进行普通攻击
4. 敌人等级同时影响战斗强度与战后经验收益
